# 🔭 JTransient

**JTransient** is a pure Java library for detecting moving objects—asteroids, comets, meteors, and artificial satellites—in aligned astronomical image sequences.

It serves as the standalone mathematical and geometrical engine behind [SpacePixels](https://github.com/ppissias/SpacePixels) (or your own custom astrophotography applications). It is completely decoupled from file I/O and UI, meaning it can easily be embedded into web backends, command-line tools, or desktop applications.

## 🌌 The Philosophy: The Opposite of Stacking

In modern astrophotography, our primary goal is almost always to create a single, deep, noise-free image of a deep-sky object. To do this, we take dozens or hundreds of individual sub-exposures and run them through stacking algorithms (like Sigma Clipping).

These algorithms are incredibly good at doing one specific thing: **throwing away anything that changes between frames**.

But what exactly are we throwing away? Those "rejected pixels" are often real, dynamic events happening above our heads. By stacking, we are literally erasing the active universe to look at the static one.

**JTransient is the exact opposite of a stacking tool.** It was built to mine your discarded data. Instead of averaging out movement to hide it, JTransient actively hunts for morphological and kinematic anomalies across your sub-exposures to reveal the hidden transients you captured while you weren't looking.

## ⚠️ Crucial Requirement: Aligned & Calibrated Inputs

JTransient does **not** load FITS files, nor does it align images. It is designed to process raw 2D pixel arrays (`short[][]`, which is the standard Java representation of a monochrome, 16-bits per pixel FITS image).

Because the engine looks for tiny pixel movements between frames, **it requires perfectly aligned (registered) and calibrated data as its input.** You should use the intermediate files generated by your existing stacking software (DeepSkyStacker, Siril, PixInsight) right before the final master stack is created, extract the 16-bit pixel data, and feed it to JTransient.

## ⚙️ How It Works (The Pipeline)

The `JTransientEngine` orchestrates a highly concurrent, 4-phase pipeline:

1. **Extraction (Image Moments):** Uses a fast Breadth-First Search (BFS) and Image Moments to extract sources from the background sky noise, classifying them as point-sources, streaks, or noise.
2. **Quality Control (MAD Statistics):** Evaluates Median Absolute Deviation (MAD) across the entire session to automatically reject anomalous frames caused by passing clouds, wind bumps, or bad focus (FWHM spikes).
3. **Master Star Map:** Compares point sources across the timeline to identify and purge stationary background stars, leaving only true transients.
4. **Kinematic & Geometric Linking:** Uses time-agnostic collinear linking to connect transient dots into confirmed tracks, applying strict photometric (brightness), morphological (size), and kinematic rhythm (speed) filters to reject artifacts.

## 🚀 Quick Start Guide

### 1. Add to your project
If you are using Gradle, include JTransient in your dependencies:
```groovy
dependencies {

```

### 2. Run the Engine
Using JTransient is incredibly simple. Wrap your raw `short[][]` arrays into `ImageFrame` objects, configure the engine, and run it!

```java
import io.github.ppissias.jtransient.engine.*;
import io.github.ppissias.jtransient.config.DetectionConfig;
import io.github.ppissias.jtransient.core.TrackLinker.Track;

import java.util.ArrayList;
import java.util.List;

public class TransientHunter {
    
    public void hunt(List<short[][]> alignedPixelArrays, List<String> filenames) throws Exception {
        
        // 1. Prepare the input frames
        List<ImageFrame> frames = new ArrayList<>();
        for (int i = 0; i < alignedPixelArrays.size(); i++) {
            frames.add(new ImageFrame(i, filenames.get(i), alignedPixelArrays.get(i)));
        }

        // 2. Configure the Engine (Easily mapped to/from JSON!)
        DetectionConfig config = new DetectionConfig();
        config.detectionSigmaMultiplier = 5.0;
        config.streakMinElongation = 4.0;
        config.maxJumpPixels = 500.0;
        // ... tune the other 30+ parameters as needed

        // 3. Execute the Pipeline
        JTransientEngine engine = new JTransientEngine();
        PipelineResult result = engine.runPipeline(frames, config);
        
        // Always shut down the engine thread pool when your app closes
        engine.shutdown();

        // 4. Handle Results
        System.out.println("Found " + result.tracks.size() + " moving targets!");
        
        for (Track track : result.tracks) {
            System.out.println("Target spans " + track.points.size() + " frames.");
            if (track.isStreakTrack) {
                System.out.println(" -> Classification: Fast-moving streak (Satellite/Meteor)");
            } else {
                System.out.println(" -> Classification: Point-source track (Asteroid/Comet)");
            }
        }
        
        // Print detailed diagnostics
        System.out.println(result.telemetry.generateReport());
    }
}
```

## 🛠️ Building JTransient from Source

JTransient uses [Gradle](https://gradle.org/) as its build system. To build the standalone `.jar` library yourself:

1. Clone the repository:
   ```bash
   git clone [https://github.com/ppissias/jtransient.git](https://github.com/ppissias/jtransient.git)
   cd jtransient
   ```

2. Build the project using the Gradle Wrapper:
   ```bash
   # On Linux/macOS
   ./gradlew build
   
   # On Windows
   gradlew.bat build
   ```

3. The compiled `.jar` file will be located in the `build/libs/` directory. You can drop this directly into the classpath of any Java application.
